## 1. Архитектура данных и ETL

### 1.1. Источники данных
| Источник | Файл | Ключевые поля |
|----------|------|---------------|
| CRM (Companies) | 01_crm_companies.csv | company_id, company_name, industry, region, employee_count |
| CRM (Transactions) | 02_crm_transactions.csv | transaction_id, company_id, product_id, total_amount, transaction_stage |
| LMS (Events) | 03_lms_events.csv | event_id, company_id, user_id, event_type, event_date |
| LMS (Courses) | 04_lms_courses.csv | course_id, course_name, category, difficulty |
| 1С (Payments) | 05_finance_payments.csv | payment_id, company_id, amount, payment_date, payment_type |
| Google Sheets | 09_products.csv | product_id, product_name, price, category |

### 1.2. Схема ETL-процесса
1. **Расчет is_active_lms:** Фильтрация событий типа 'login' за последние 90 дней → агрегация по company_id
2. **Расчет payment_stability и avg_check:** Фильтрация сделок со статусом 'closed_won' → агрегация по company_id
3. **Нормализация avg_check:** avg_check / max(avg_check) по всем компаниям
4. **Расчет readiness_score:** is_active_lms × 0.3 + payment_stability × 0.4 + norm_avg_check × 0.3
