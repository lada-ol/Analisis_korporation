# Проект витрины данных: company_readiness_for_pro

## Группа А. Оценка потенциала подписки "Pro"

### 1. Цель создания витрины

Витрина данных предназначена для оценки готовности B2B-клиентов к апсейлу на подписку "Pro". На основе витрины строится вероятностный прогноз числа продаж и ожидаемой выручки в следующем квартале.

**Ключевые бизнес-вопросы, на которые отвечает витрина:**
- Какие компании являются потенциальными клиентами для подписки Pro?
- Какова интегральная оценка готовности каждой компании?
- Каковы факторы, влияющие на готовность (активность, платежи)?

### 2. Уровень агрегации

**Уровень: Компания (одна строка = одна компания)**

**Обоснование:** 
- Прогноз продаж подписки Pro делается на уровне компании (покупает компания, а не отдельный сотрудник)
- Критерии готовности оцениваются на уровне компании (суммарная активность всех сотрудников, общая история платежей)
- Уровень компании позволяет легко объединять данные из разных источников

### 3. Структура витрины

| Поле | Тип | Источник | Описание | Способ расчета |
|------|-----|----------|----------|----------------|
| `company_id` | int | `01_crm_companies.csv` | Уникальный идентификатор компании | Прямое копирование |
| `company_name` | string | `01_crm_companies.csv` | Название компании | Прямое копирование |
| `industry` | string | `01_crm_companies.csv` | Отрасль деятельности | Прямое копирование |
| `region` | string | `01_crm_companies.csv` | Федеральный округ | Прямое копирование |
| `employee_count` | int | `01_crm_companies.csv` | Количество сотрудников | Прямое копирование |
| `status` | string | `01_crm_companies.csv` | Статус компании (active/lead/inactive) | Прямое копирование |
| `is_active_lms` | int (0/1) | `03_lms_events.csv` | Флаг активности в LMS за последние 90 дней | 1 если есть хотя бы один login за последние 90 дней от cutoff_date, иначе 0 |
| `login_count_90d` | int | `03_lms_events.csv` | Количество логинов за последние 90 дней | Подсчет событий event_type='login' за последние 90 дней |
| `payment_stability` | float (0-1) | `05_finance_payments.csv` | Коэффициент стабильности платежей | successful_payments / total_payments (только payment_type='payment', amount>0). Для компаний без платежей = 0 |
| `total_payments_count` | int | `05_finance_payments.csv` | Общее количество платежей | Подсчет записей с payment_type='payment', amount>0 |
| `avg_check` | float | `02_crm_transactions.csv` | Средний чек компании по успешным сделкам | AVG(total_amount) WHERE transaction_stage='closed_won' |
| `total_revenue` | int | `02_crm_transactions.csv` | Общая выручка компании по успешным сделкам | SUM(total_amount) WHERE transaction_stage='closed_won' |
| `readiness_score` | float (0-1) | Расчетное поле | Интегральная оценка готовности к Pro | Взвешенная сумма нормализованных метрик (см. формулу ниже) |
| `ready_for_pro` | int (0/1) | Расчетное поле | Бинарный флаг готовности | 1 если readiness_score >= порогового значения (определяется в Пр-2) |

### 4. Формула расчета readiness_score

```python
# Исходные метрики
is_active_lms     # 0 или 1
payment_stability # от 0 до 1 (уже нормализована)
avg_check         # от 0 до max_avg_check

# Нормировка avg_check
norm_avg_check = avg_check / max(avg_check)  # max берется по всем компаниям

# Весовые коэффициенты (согласно ТЗ из Пр-1)
# Веса обоснованы: платежи важнее активности (0.4 > 0.3),
# средний чек - прокси размера компании и потенциала
readiness_score = (
    is_active_lms * 0.3 + 
    payment_stability * 0.4 + 
    norm_avg_check * 0.3
)
