# Метаданные источников данных

## ИСТОЧНИК: 01_crm_companies.csv

### 1. Идентификация
- **Система:** CRM (Система управления взаимоотношениями с клиентами)
- **Тип источника:** Справочник (основной справочник компаний-клиентов)
- **Имя файла:** 01_crm_companies.csv

### 2. Назначение
Содержит карточки компаний-клиентов B2B-сегмента. Используется как основной справочник для идентификации компаний, их базовых атрибутов (название, ИНН, отрасль, регион, размер) и текущего статуса в воронке продаж (active/lead).

### 3. Владелец
- **Подразделение:** Департамент продаж (B2B-блок)
- **Контактное лицо:** Козлов Дмитрий Викторович (руководитель направления B2B), менеджеры Иванов И., Петрова А., Смирнов С.

### 4. Структура
- **Количество полей:** 8
- **Перечень полей с типами:**
    - `company_id` (int): Уникальный идентификатор компании.
    - `company_name` (string): Название компании.
    - `inn` (int): ИНН компании (может быть пропущен).
    - `industry` (string): Отрасль деятельности компании.
    - `region` (string): Федеральный округ, в котором зарегистрирована компания.
    - `employee_count` (int): Количество сотрудников.
    - `created_date` (date, format YYYY-MM-DD): Дата создания карточки в CRM.
    - `status` (string): Статус компании (active - активный, lead - лид, inactive - неактивный).
- **Количество записей:** 40 (от company_id 1001 до 1040)

### 5. Временные характеристики
- **Период данных:** с 2022-01-10 по 2024-11-15 (исходя из `created_date`).
- **Частота обновления:** Раз в неделю, скриптом. Данные актуальны на момент выгрузки.

### 6. Предварительная оценка качества
- **Замеченные аномалии (при беглом просмотре):**
    - Присутствуют пропуски в поле `inn` (например, для company_id 1003).
    - Компания "ООО ""Ромашка""" (1001) и "ООО ""Ромашка-Логистик""" (1004) имеют одинаковый ИНН (7701123456), что указывает на возможную связь или проблему с дублированием данных.

### 7. Связи с другими источниками
- **Связан с:**
    - `02_crm_transactions.csv`
    - `03_lms_events.csv`
    - `05_finance_payments.csv`
    - `06_support_tickets.csv`
    - `07_marketing_leads.csv`
    - `13_company_history.csv`
- **По каким полям:**
    - `company_id` (связь с транзакциями, событиями, платежами, тикетами, историей).
    - `company_name` (связь с маркетинговыми лидами, неформально, так как ИНН часто отсутствует).

### 8. Ограничения
Данные по ИНН могут быть неполными и невалидными, так как не проверяются отделом продаж. Не следует использовать `inn` как единственный ключ для связывания с другими источниками без валидации и обработки пропусков.

## ИСТОЧНИК: 02_crm_transactions.csv

### 1. Идентификация
- **Система:** CRM (Система управления взаимоотношениями с клиентами)
- **Тип источника:** Операционная (фиксация сделок и транзакций)
- **Имя файла:** 02_crm_transactions.csv

### 2. Назначение
Содержит данные о сделках и транзакциях с клиентами. Используется для анализа продаж, расчета выручки, отслеживания этапов сделки и оценки эффективности менеджеров.

### 3. Владелец
- **Подразделение:** Департамент продаж (B2B-блок)
- **Контактное лицо:** Козлов Дмитрий Викторович, менеджеры Иванов И., Петрова А., Смирнов С.

### 4. Структура
- **Количество полей:** 9
- **Перечень полей с типами:**
    - `transaction_id` (string): Уникальный идентификатор транзакции.
    - `company_id` (int): Идентификатор компании-клиента.
    - `transaction_date` (date, format YYYY-MM-DD): Дата совершения сделки.
    - `product_id` (string): Идентификатор продукта/услуги.
    - `quantity` (int): Количество единиц продукта.
    - `unit_price` (int): Цена за единицу.
    - `total_amount` (int): Общая сумма сделки.
    - `transaction_stage` (string): Этап сделки (closed_won, negotiation, lead).
    - `manager` (string): Менеджер, ответственный за сделку.
    - `probability` (int): Вероятность закрытия сделки в процентах.
- **Количество записей:** 42

### 5. Временные характеристики
- **Период данных:** с 2024-12-10 по 2025-03-31 (исходя из `transaction_date`).
- **Частота обновления:** Раз в неделю, скриптом.

### 6. Предварительная оценка качества
- **Замеченные аномалии (при беглом просмотре):**
    - Присутствуют транзакции с `transaction_stage`, отличным от 'closed_won', которые могут быть нефинальными.
    - Сделки на сумму 35000 (D014) и 50000 (D018) могут содержать аномальные значения, которые стоит проверить (например, продукт P099 с ценой 50000, которого нет в каталоге курсов).
    - В поле `unit_price` есть несоответствие: для product_id P004 цена варьируется (15000, 11667, 25000, 13000), что требует проверки.

### 7. Связи с другими источниками
- **Связан с:**
    - `01_crm_companies.csv`
    - `09_products.csv` (для получения названия продукта)
    - `05_finance_payments.csv` (для сверки оплат)
- **По каким полям:**
    - `company_id` (с компанией)
    - `product_id` (с продуктами)

### 8. Ограничения
Данные отражают этап сделки, а не фактическое поступление денег. Для расчета реальной выручки требуется сопоставление с финансовой системой (1С). Нет прямой связи с маркетинговыми кампаниями, которые привели к сделке.

## ИСТОЧНИК: 03_lms_events.csv

### 1. Идентификация
- **Система:** LMS (Платформа для обучения, ООО "ЛМС-Технологии")
- **Тип источника:** Логи (события пользователей в системе)
- **Имя файла:** 03_lms_events.csv

### 2. Назначение
Фиксирует события пользователей в LMS (логины, начало и завершение уроков). Используется для анализа активности клиентов, глубины вовлеченности в обучение и завершаемости курсов.

### 3. Владелец
- **Подразделение:** Платформа (ООО "ЛМС-Технологии")
- **Контактное лицо:** Новиков Александр Александрович (руководитель)

### 4. Структура
- **Количество полей:** 7
- **Перечень полей с типами:**
    - `event_id` (string): Уникальный идентификатор события.
    - `company_id` (int): Идентификатор компании-клиента.
    - `user_id` (int): Уникальный идентификатор пользователя в LMS.
    - `course_id` (string): Идентификатор курса.
    - `event_type` (string): Тип события (login, lesson_start, lesson_complete).
    - `event_date` (datetime, format YYYY-MM-DD HH:MM:SS): Дата и время события.
    - `completion_pct` (int, nullable): Процент завершения урока (только для event_type='lesson_start').
- **Количество записей:** 50

### 5. Временные характеристики
- **Период данных:** с 2025-01-20 по 2025-03-31 (исходя из `event_date`).
- **Частота обновления:** Раз в сутки, выгрузка в CSV.

### 6. Предварительная оценка качества
- **Замеченные аномалии (при беглом просмотре):**
    - Поле `completion_pct` имеет много пропусков (NULL), которые соответствуют событиям типа 'login'.
    - Встречаются странные значения `completion_pct` (например, 5, 10, 15, 25, 30), которые могут быть не в процентах, а в минутах или других единицах. Это требует проверки и, скорее всего, преобразования.
    - Формат даты в некоторых строках (E0022) не содержит времени.

### 7. Связи с другими источниками
- **Связан с:**
    - `01_crm_companies.csv`
    - `04_lms_courses.csv`
    - `05_finance_payments.csv` (косвенно, через company_id)
- **По каким полям:**
    - `company_id` (с компанией)
    - `course_id` (с курсами)
    - `user_id` (связь с конкретными сотрудниками компании, но не с CRM)

### 8. Ограничения
Нет прямой связи между `user_id` в LMS и конкретными сотрудниками в CRM. Данные события не показывают, сколько времени пользователь потратил на урок. События хранятся только 2 года, после чего архивируются.

## ИСТОЧНИК: 04_lms_courses.csv

### 1. Идентификация
- **Система:** LMS (Платформа для обучения, ООО "ЛМС-Технологии")
- **Тип источника:** Справочник (каталог курсов)
- **Имя файла:** 04_lms_courses.csv

### 2. Назначение
Содержит каталог всех курсов, доступных на платформе, с их описанием, категорией, сложностью и длительностью. Используется для обогащения событий обучения и для анализа популярности курсов.

### 3. Владелец
- **Подразделение:** Методический отдел (в составе B2B-блока или B2C-сегмента). По информации из файла `10_info.txt`, точный владелец не определен ("вроде методисты, но точно не знаю").
- **Контактное лицо:** Кузнецова Елена (руководитель продукта Pro) или Новиков А.А. (платформа) как технический владелец.

### 4. Структура
- **Количество полей:** 6
- **Перечень полей с типами:**
    - `course_id` (string): Уникальный идентификатор курса.
    - `course_name` (string): Название курса.
    - `category` (string, nullable): Категория курса (может быть пропущена).
    - `difficulty` (string): Уровень сложности (beginner, intermediate, advanced).
    - `duration_hours` (int): Длительность курса в часах.
    - `description` (string): Описание курса.
- **Количество записей:** 8

### 5. Временные характеристики
- **Период данных:** Актуальный справочник на момент выгрузки.
- **Частота обновления:** По мере добавления/изменения курсов методическим отделом. Технически может обновляться разработчиками.

### 6. Предварительная оценка качества
- **Замеченные аномалии (при беглом просмотре):**
    - Есть пропуск в поле `category` для курса `C007` (Продуктовая аналитика), что делает его сложным для группировки.
    - Данные представлены на русском языке, что может быть неудобно для машинной обработки.

### 7. Связи с другими источниками
- **Связан с:**
    - `03_lms_events.csv`
- **По каким полям:**
    - `course_id` (с событиями в LMS)

### 8. Ограничения
Справочник является статичным и не содержит информации о версиях курсов или датах их добавления/удаления. Это может быть проблемой для анализа исторических данных, если курс был изменен.
