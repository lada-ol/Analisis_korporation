## Метрика 1: Активный клиент (Active Customer)

### Определения из разных источников

| Источник | Определение | Откуда взято |
|----------|-------------|---------------|
| Финансы (1С) | Платежи за последние 90 дней | 10_info.txt |
| Продукт (LMS) | Логины за последние 30 дней | 10_info.txt |
| Продажи (CRM) | Статус 'active' или 'lead' | 10_info.txt |

### Количественный анализ

- Финансовое определение (платежи за 90 дней): ~15-20 компаний
- Продуктовое определение (логины за 30 дней): ~8-12 компаний
- Продажное определение (статус 'active/lead'): ~35 компаний
- Пересечение всех трех определений: ~5-8 компаний
- Компании, которые платят, но не заходят в LMS (только в финансах): ~5-10 компаний

### Интерпретация

Существует группа компаний, которые платят, но не заходят в LMS (~5-10 компаний). Это потенциальная группа риска оттока — клиенты забыли про продукт, но продолжают платить. Продажное определение дает наибольшее количество активных клиентов, но оно наименее надежно, так как статус может быть устаревшим.

### Предложение единого определения

| Поле | Значение |
|------|----------|
| **Метрика** | Активный клиент |
| **Определение** | Клиент считается активным, если за последние 90 дней компания осуществила хотя бы один платеж (payment_type='payment', amount>0) ИЛИ хотя бы один сотрудник компании совершал вход в LMS (event_type='login') |
| **Обоснование** | Комбинация финансового и продуктового критерия позволяет выявить как платящих, так и использующих продукт клиентов, а также идентифицировать группу риска (платят, но не используют) |
| **Алгоритм расчета** | Объединение множеств: (платежи за 90 дней) ∪ (логины за 90 дней) |
| **Владелец** | Аналитический департамент |

## Метрика 2: Выручка от клиента (Customer Revenue)

### Определения из разных источников

| Источник | Определение | Откуда взято |
|----------|-------------|---------------|
| Финансы (1С) | Сумма платежей минус возвраты | Протокол совещания (10_info.txt) |
| Продажи (CRM) | Сумма закрытых сделок (closed_won) | Протокол совещания (10_info.txt) |

### Количественный анализ

- Выручка по финансам (1С) за Q1 2025: ~3-4 млн рублей
- Выручка по продажам (CRM) за Q1 2025: ~4-5 млн рублей
- Расхождение: ~20-30%
- Компании, по которым есть расхождения: ~5-10 компаний

### Интерпретация

Выручка по продажам (CRM) систематически выше, чем по финансам (1С), так как сделки могли быть закрыты, а деньги еще не поступили. Существуют компании, которые платят, но не имеют closed_won сделок (возможно, оплата по счету без формальной сделки в CRM).

### Предложение единого определения

| Поле | Значение |
|------|----------|
| **Метрика** | Выручка от клиента |
| **Определение** | Для целей финансовой отчетности: сумма фактических платежей минус возвраты. Для целей оценки продаж: сумма закрытых сделок |
| **Обоснование** | Финансовое определение объективно для денежного учета. Продажное определение необходимо для мотивации менеджеров и оперативного управления |
| **Алгоритм расчета** | Финансы: SUM(amount) WHERE payment_type='payment' - SUM(ABS(amount)) WHERE payment_type='refund' |
| **Владелец** | Департамент финансов |

## Метрика 3: Средний чек (Average Check)

### Определения из разных источников

| Источник | Определение | Откуда взято |
|----------|-------------|---------------|
| CRM | Средняя сумма успешной сделки (closed_won) | 02_crm_transactions.csv |
| Финансы (1С) | Средняя сумма платежа | 05_finance_payments.csv |

### Количественный анализ

- Средний чек по CRM (closed_won сделки): ~120 000 рублей
- Средний платеж по 1С: ~100 000 рублей
- Расхождение: ~20%
- Коэффициент асимметрии: >2 (распределение сильно скошено)
- Медианный чек: ~70-80 тыс. рублей (значительно ниже среднего)

### Интерпретация

Средний чек по сделкам (CRM) выше, чем средний платеж (финансы), так как крупные сделки могут оплачиваться в рассрочку (один чек → несколько платежей). Распределение сильно скошено — есть несколько очень крупных сделок, которые "тянут" среднее значение вверх. Типичный чек/платеж (медиана) значительно ниже среднего арифметического.

### Предложение единого определения

| Поле | Значение |
|------|----------|
| **Метрика** | Средний чек |
| **Определение** | Для целей прогнозирования выручки и оценки потенциала клиента используется средняя сумма одной успешной сделки (closed_won) в CRM |
| **Обоснование** | Средний чек по сделкам лучше отражает ценность продукта для клиента, так как одна сделка может включать несколько продуктов или подписку на период. Средний платеж может быть искажен из-за рассрочек |
| **Алгоритм расчета** | AVG(total_amount) FROM transactions WHERE transaction_stage='closed_won' |
| **Владелец** | Аналитический департамент |

## Влияние на бизнес

Внедрение единых определений позволит:
1. Устранить расхождения в отчетности разных департаментов
2. Корректно оценивать группу риска (клиенты, которые платят, но не используют продукт)
3. Строить точные прогнозы выручки на основе согласованных метрик
4. Сравнивать эффективность разных каналов привлечения клиентов

## Следующие шаги

1. Утвердить предложенные определения на ближайшем совещании с руководителями департаментов
2. Внести определения в корпоративный глоссарий
3. Настроить регулярный расчет метрик в витрине данных company_readiness_for_pro
4. Провести обучение команд единым подходам к расчету метрик
