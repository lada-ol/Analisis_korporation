## 2. Назначение и бизнес-контекст

Данные собираются для ведения единого реестра компаний-клиентов B2B-сегмента. Карточка компании создается в момент первого контакта с потенциальным клиентом и обновляется при изменении информации о компании. Данные используются для идентификации клиентов, анализа клиентской базы по отраслям, регионам и размеру, а также для связывания с другими системами (транзакции, платежи, события в LMS).

## 6. Правила трансформации (ETL)

| Действие | Описание | Код |

## 7. Ограничения использования

| Ограничение | Пояснение | Рекомендация |
|-------------|-----------|--------------|
| ИНН не является надежным ключом | 20% пропусков, есть дубликаты | Использовать company_id как основной ключ |
| Статус может быть устаревшим | Обновляется раз в неделю | Учитывать задержку при анализе |
| Нет валидации ИНН | Менеджеры не проверяют ИНН | Не использовать для юридически значимых действий |
| Данные нереального времени | Выгрузка раз в неделю | Не использовать для оперативной аналитики |
