## 2. Назначение и бизнес-контекст

Данные фиксируют все финансовые транзакции: поступления платежей от клиентов и возвраты. Являются источником истины для финансового учета. Используются для расчета реальной выручки, оценки стабильности платежей, выявления проблемных клиентов и финансового планирования.

## 6. Правила трансформации (ETL)

| Действие | Описание | Код |
|----------|----------|-----|
| Загрузка | Чтение CSV файла | `df = pd.read_csv('05_finance_payments.csv')` |
| Преобразование дат | КРИТИЧЕСКИ ВАЖНО: формат ДД.ММ.ГГГГ | `df['payment_date'] = pd.to_datetime(df['payment_date'], format='%d.%m.%Y')` |
| Фильтрация платежей | Только payment_type='payment', amount>0 | `payments_clean = df[(df['payment_type'] == 'payment') & (df['amount'] > 0)]` |
| Подсчет количества платежей | Группировка по company_id | `payment_stats = payments_clean.groupby('company_id').agg(total_payments_count=('payment_id', 'count'))` |
| Обработка возвратов (отдельно) | Для информации и анализа | `refunds = df[df['payment_type'] == 'refund']` |

## 7. Ограничения использования

| Ограничение | Пояснение | Рекомендация |
|-------------|-----------|--------------|
| Даты требуют преобразования | Формат ДД.ММ.ГГГГ не совместим с pandas по умолчанию | Обязательно использовать format='%d.%m.%Y' |
| Не учитывать возвраты как платежи | Отрицательные суммы искажают агрегаты | Фильтровать payment_type='payment' |
| Нет связи с продуктами | Только сумма без детализации | Использовать CRM для привязки к продуктам |
| Возможны задержки платежей | Дата платежа ≠ дате сделки | Учитывать временные лаги при анализе |
| Не использовать как единственный источник выручки без учета возвратов | Может давать завышенные показатели | Всегда фильтровать payment_type |
