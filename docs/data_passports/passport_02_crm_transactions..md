## 2. Назначение и бизнес-контекст

Данные фиксируют все сделки и транзакции с клиентами B2B-сегмента. Каждая сделка отражает продажу продукта или услуги компании. Данные используются для анализа продаж, расчета выручки, отслеживания этапов сделки, оценки эффективности менеджеров и построения прогнозов.

| Поле | Тип | Описание | Пример | Обязательность | Примечания |

## 6. Правила трансформации (ETL)

| Действие | Описание | Код |
|----------|----------|-----|
| Загрузка | Чтение CSV файла | `df = pd.read_csv('02_crm_transactions.csv')` |
| Преобразование дат | Конвертация в datetime | `df['transaction_date'] = pd.to_datetime(df['transaction_date'])` |
| Фильтрация успешных | Только closed_won для avg_check | `closed_won = df[df['transaction_stage'] == 'closed_won']` |
| Проверка целостности | Верификация total_amount | `df['calculated_amount'] = df['quantity'] * df['unit_price']` |
| Агрегация среднего чека | Group by company_id | `avg_check = closed_won.groupby('company_id')['total_amount'].mean()` |
| Агрегация выручки | Group by company_id | `total_revenue = closed_won.groupby('company_id')['total_amount'].sum()` |

## 7. Ограничения использования

| Ограничение | Пояснение | Рекомендация |
|-------------|-----------|--------------|
| Суммы сделок ≠ фактическим платежам | Сделка может быть закрыта, но деньги не поступили | Для реальных денег использовать 05_finance_payments.csv |
| Статус 'negotiation/lead' не гарантирует сделку | Только closed_won = реальная продажа | При расчетах использовать только closed_won |
| Задержка обновления | Данные обновляются раз в неделю | Учитывать лаг в оперативной аналитике |
| unit_price не является стандартной ценой | Цены могут варьироваться в зависимости от сделки | Использовать total_amount, а не unit_price |
| Отсутствует closed_lost | Невозможно рассчитать полную конверсию | Учитывать limitation при анализе воронки |
