## 2. Назначение и бизнес-контекст

Данные фиксируют все события пользователей в LMS: логины, начало уроков, завершение уроков. Используются для анализа активности клиентов, глубины вовлеченности в обучение, оценки "здоровья" клиентской базы и выявления групп риска оттока.

## 3. Структура данных

| Поле | Тип | Описание | Пример | Обязательность | Примечания |
|------|-----|----------|--------|----------------|------------|
| event_id | string | Уникальный идентификатор события | E0001 | Да | Первичный ключ |
| company_id | int | Идентификатор компании | 1001 | Да | Внешний ключ к companies |
| user_id | int | Идентификатор пользователя в LMS | 1001001 | Да | Не связан с CRM |
| course_id | string | Идентификатор курса | C001 | Да | Связь с courses |
| event_type | string | Тип события | login | Да | login, lesson_start, lesson_complete |
| event_date | datetime | Дата и время события | 2025-02-10 9:15:00 | Да | Формат YYYY-MM-DD HH:MM:SS |
| completion_pct | int | Процент завершения урока | 25 | Нет | Только для lesson_start |

## 4. Происхождение и обновление

| Параметр | Значение |

| Период хранения | 2 года, после чего архивируется |

## 5. Качество данных

| Показатель | Значение |
|------------|----------|
| Полнота данных | 100% для ключевых полей; completion_pct: 40% |
| Уникальность ключей | event_id: уникален, дубликатов нет |
| Выявленные проблемы | 1. completion_pct похож на минуты (5,10,15,25,30), а не проценты. 2. Нет связи user_id с CRM. 3. Разные форматы event_date (E0022 без времени). |

## 6. Правила трансформации (ETL)

| Действие | Описание | Код |
|----------|----------|-----|
| Загрузка | Чтение CSV файла | `df = pd.read_csv('03_lms_events.csv')` |
| Преобразование дат | Извлечение даты из строки с временем | `df['event_date'] = pd.to_datetime(df['event_date'].str.split().str[0])` |
| Фильтрация логинов | Только event_type='login' | `logins = df[df['event_type'] == 'login']` |
| Фильтрация по периоду | За последние 90 дней от cutoff_date | `logins_90d = logins[logins['event_date'] >= lookback_90]` |
| Подсчет активности | Группировка по company_id | `lms_activity = logins_90d.groupby('company_id').agg(login_count_90d=('event_id', 'count'))` |
| Создание флага | 1 если есть логины, иначе 0 | `lms_activity['is_active_lms'] = (lms_activity['login_count_90d'] > 0).astype(int)` |

## 7. Ограничения использования

| Ограничение | Пояснение | Рекомендация |
|-------------|-----------|--------------|
| Нет связи user_id с CRM | Нельзя идентифицировать конкретных сотрудников компании | Агрегация только на уровне company_id |
| completion_pct ненадежен | Возможно, это минуты, а не проценты | Не использовать для расчетов в Группе А |
| Ограниченный период хранения | События хранятся только 2 года | Учитывать при историческом анализе |
| Только логин как прокси активности | Не учитывается реальное обучение | Дополнять другими метриками при возможности |
| Разные форматы дат | Может вызвать ошибки при парсинге | Обязательная предобработка |
